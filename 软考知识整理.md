## 1、码制


### 表示
| 码制 | 定点整数                  | 定点小数                        | 数码个数 | 用途 |
| :- | :-------------------- | :-------------------------- | :--- | :--- |
| 原码 | -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1) | -(1-2⁻⁽ⁿ⁻¹⁾) ~ +(1-2⁻⁽ⁿ⁻¹⁾) | 2ⁿ-1 | 简单表示，乘除法 |
| 反码 | -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1) | -(1-2⁻⁽ⁿ⁻¹⁾) ~ +(1-2⁻⁽ⁿ⁻¹⁾) | 2ⁿ-1 | 原码与补码的中间过渡 |
| 补码 | -2ⁿ⁻¹ ~ +(2ⁿ⁻¹-1)     | -1 ~ +(1-2⁻⁽ⁿ⁻¹⁾)           | 2ⁿ   | 计算机内部存储和运算整数（最常用） |
| 移码 | -2ⁿ⁻¹ ~ +(2ⁿ⁻¹-1)     | -1 ~ +(1-2⁻⁽ⁿ⁻¹⁾)           | 2ⁿ   | 浮点数的阶码表示，便于比较大小 |

计算机内部用补码存整数，因为加减法统一、0 唯一。例：-3 的补码计算过程：原码 10000011 → 反码 11111100 → 补码 11111101。
### 考点
- 补码范围不对称：最小负数的绝对值比最大正数大1
如8位补码：-128 ~ +127
- 补码中 -2ⁿ⁻¹ 没有原码和反码
10000000（8位）在补码中表示-128，在原码中是非法的 
- 移码与补码的关系：符号位相反，数值位相同 
- 定点小数 -1 的补码：1.000...0（纯小数，整数部分为1表示负数）
## 2、浮点数
### 格式
阶码决定范围，尾数决定精度。越长，范围越大，精度越高。
### 计算过程
1. 解析浮点数：从存储的二进制位中提取符号位、指数位和尾数位。
2. 处理指数：对于加减法，需要对齐指数；对于乘除法，指数相加或相减。
3. 处理尾数：将尾数对齐或相乘，并考虑进位或舍入。
4. 规格化：确保计算结果符合 IEEE 754 规范。
5. 编码为浮点数：将最终结果重新组织成二进制形式。

**过程：对阶 -> 尾数计算 -> 格式化**

**对阶：小数向大数看齐，尾数右移**

## 4、流水线执行时间计算

- **理论公式**：建立时间 + (指令条数 - 1) × 流水线周期
- **实际公式**：指令段数 × 流水线周期 + (指令条数 - 1) × 流水线周期
- **吞吐率** = 指令条数 / 流水线执行时间
- **最大吞吐率** = 流水线周期的导数

## 10、数据库封锁协议

- **共享锁（S锁）**：允许多个事务同时读取同一数据项，但不允许写操作。
- **排他锁（X锁）**：只允许一个事务对数据项进行读取和修改，其他事务必须等待该事务释放锁后才能访问。

## 11、层次化网络

- **汇聚层**：承上启下，连接接入层和核心层，负责数据链路汇总、跨子网路由、访问控制与策略实施以及广播域隔离。主要设备有三层交换机和路由器，有时结合防火墙。
- **接入层**：主要设备有接入交换机和无线接入点（AP），用于连接终端设备，提供基本的网络接入功能。

## 12、加密算法分类、优缺点和应用场景

加密算法主要分为三类：

- **对称加密**：使用单一密钥进行加密和解密，速度快、资源消耗少，适合大数据量加密，但密钥管理复杂，常用于文件加密和磁盘加密。
- **非对称加密**：使用公钥加密、私钥解密，安全性高，但计算速度慢，常与对称加密结合用于数字签名和安全通信，如 HTTPS 协议。
- **哈希算法**：生成固定长度的摘要。

## 13、HTTPS 通信加解密流程

HTTPS 通信的加解密流程主要分为两个阶段：**握手阶段**和**数据传输阶段**。

在握手阶段，客户端向服务器发送请求，服务器回应数字证书和公钥。客户端验证证书后，生成随机对称密钥，用服务器公钥加密后发送给服务器。服务器用私钥解密得到对称密钥。

## 14、McCabe 复杂度

McCabe 复杂度是一种用于衡量代码逻辑复杂度的指标，基于图论中的环路复杂度概念。

**计算公式**：复杂度 = 边数 - 节点数 + 2

它通过量化代码中所有可能的独立路径数量，帮助开发者
评估代码复杂性，进而改善代码质量

## 21. 总线
### 分类
┌─────────────────────────────────────┐
│           系统总线（外部总线）          │  ← 连接各大部件
│    CPU ←→ 主存 ←→ I/O接口 ←→ 外设    │
└─────────────────────────────────────┘
│
┌────┴────┐
▼         ▼
┌───────┐  ┌─────────┐
│数据总线│  │地址总线 │  │控制总线│
└───────┘  └─────────┘


| 总线类型     | 连接范围               | 功能              |
| :------- | :----------------- | :-------------- |
| **系统总线** | CPU、主存、I/O设备       | 计算机各大部件之间的信息传输  |
| 内部总线     | CPU内部（ALU、寄存器、控制器） | 芯片内部数据传送        |
| I/O总线    | 外设与主机              | 如USB、PCIe、SATA等 |
| 通信总线     | 计算机系统之间            | 如以太网、RS-232     |
### 系统总线的组成
| 组成           | 功能                    |
| :----------- | :-------------------- |
| **数据总线（DB）** | 双向传输数据，宽度决定一次能传输的数据位数 |
| **地址总线（AB）** | 单向传输（CPU发出），宽度决定寻址能力  |
| **控制总线（CB）** | 传输控制信号（读/写、中断、DMA请求等） |

### 系统总线定义
系统总线：连接计算机主要功能部件（CPU、主存、I/O设备）的公共信息通道
系统总线上的典型设备：
├── CPU（通过总线接口）
├── 主存/内存（DRAM）
├── I/O接口电路
│   ├── 显卡（显示适配器）
│   ├── 声卡
│   ├── 网卡
│   └── 其他外设控制器
└── DMA控制器、中断控制器等（作为I/O部件）

* 主存是核心存储部件
* 外设部件通过I/O接口连接到系统总线
* 系统总线正是用来连接这两大类部件的桥梁

## 排序算法
### 时间复杂度
| 排序算法     | 平均时间         | 最好         | 最坏         | 说明 |
| :------- | :----------- | :---------- | :---------- | :--- |
| **快速排序** | **O(n log n)** | O(n log n)  | O(n²)       | 最常用，但最坏情况退化 |
| 归并排序     | O(n log n)   | O(n log n)  | O(n log n)  | 稳定但需O(n)额外空间 |
| 堆排序      | O(n log n)   | O(n log n)  | O(n log n)  | 原地排序，无最坏退化 |
| 冒泡排序     | O(n²)        | O(n)        | O(n²)       | 最好情况O(n)已有序 |
| 简单选择排序   | O(n²)        | O(n²)       | O(n²)       | 始终O(n²)，不稳定 |
### 空间复杂度
| 排序算法     | 辅助空间复杂度         | 说明                      |
| :------- | :-------------- | :---------------------- |
| **归并排序** | **O(n)**        | ✅ **最多**，需要额外数组存储合并结果   |
| 快速排序     | O(log n) ~ O(n) | 递归栈空间，平均O(log n)，最坏O(n) |
| 堆排序      | O(1)            | 原地排序，仅需常数额外空间           |
| 冒泡排序     | O(1)            | 原地排序，仅需临时变量             |
### 稳定性
| 排序算法     | 稳定性       | 判断                |
| :------- | :-------- | :---------------- |
| **冒泡排序** | **稳定** ✅  | 相等元素不交换，相对位置不变    |
| 快速排序     | **不稳定** ❌ | 分区交换可能改变相等元素相对位置  |
| 堆排序      | **不稳定** ❌ | 建堆和调整过程会改变元素相对位置  |
| 简单选择排序   | **不稳定** ❌ | 选择最小元素交换时可能改变相对位置 |
